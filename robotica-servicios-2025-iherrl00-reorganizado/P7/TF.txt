# Práctica: Transformaciones (TF) con Detección de Humanos en ROS 2

Este documento contiene todos los comandos necesarios para configurar, compilar y ejecutar la práctica de transformaciones de coordenadas (TF) con detección facial en ROS 2 Jazzy.

## Descripción del Proyecto
Este proyecto implementa un sistema de transformaciones de coordenadas en ROS 2 que:
- Publica una transformación estática entre el portátil (`base_link`) y la cámara (`camera_frame`)
- Detecta caras humanas usando OpenCV
- Calcula y publica la posición del humano detectado respecto al portátil
- Visualiza todo el árbol de transformaciones en RViz2

Frames utilizados:
- base_link → origen del sistema (portátil)
- camera_frame → posición de la cámara
- human_in_camera → posición del humano detectado

## Visualización con RViz2

### Abrir RViz2

En una nueva terminal:

```bash
cd ~/robotica-servicios-2025-iherrl00/ros2_ws/
source install/setup.bash
rviz2
```

### Configuración de RViz2

1. Cambiar Fixed Frame:
   - Panel izquierdo → Global Options
   - Fixed Frame: cambiar a `base_link`

2. Añadir display TF:
   - Click en botón "Add" (abajo izquierda)
   - Seleccionar pestaña "By display type"
   - Buscar y seleccionar "TF"
   - Click "OK"

3. Configurar visualización del TF:
   - Expandir sección "TF" en el panel izquierdo
   - Marcar ✓ "Show Names"
   - Marcar ✓ "Show Axes"
   - Ajustar "Marker Scale" a 0.5 o 1.0

4. Añadir imagen de cámara (opcional):
   - Click en "Add"
   - Seleccionar "Image"
   - En "Topic" → seleccionar `/camera/image_raw` o `/image_raw`

### Ver las transformaciones estáticas


```bash

```

```bash

```

### Frame 'human_in_camera' no aparece

El frame solo aparece cuando se detecta una cara. Verifica los logs:

```bash
ros2 topic hz /tf
```

## Conceptos Clave

### REPs (ROS Enhancement Proposals)

Este proyecto sigue las convenciones definidas en:
- REP 103: Unidades SI y sistema de coordenadas (X→adelante, Y→izquierda, Z→arriba)
- REP 105: Nombres estándar de frames (base_link, camera_frame)
- REP 120: Sistema TF2 y representación temporal
- REP 2011: Convenciones de TF2 en ROS 2

### Transformaciones Estáticas vs Dinámicas

- Estáticas: Relación fija entre frames (ej: base_link → camera_frame)
- Dinámicas: Relación que cambia en el tiempo (ej: camera_frame → human_in_camera)
